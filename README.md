# LastEcho

An interactive globe-based platform for visualizing endangered language data and coordinating structured outreach to institutions that can help document and preserve them before they're lost.

The public frontend lets anyone explore language vitality across time (2000–2050) on a 3D globe. Behind the scenes, a backend triage sweep automatically ranks the most at-risk languages, drafts personalized outreach emails via Claude AI, and queues them for human review before sending. When institutions don't reply, the system escalates through a three-tier ladder: local → continental → global.

| Layer | Tech |
|---|---|

## Getting Started

### Prerequisites

- Node.js 18+
- Python 3.12+

### Backend

```bash
cd api
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Generate the language dataset (only needed once, or after data updates)
python scripts/build_data.py

uvicorn app.main:app --reload --port 8000
```

### Frontend

```bash
cd client
npm install
npm run dev
```

- App: http://localhost:5173
- API docs: http://localhost:8000/docs
- Admin panel: http://localhost:5173/about/admin

## Environment Variables

Copy `api/.env.example` to `api/.env` and fill in what you need:

```bash
# Admin auth — required for admin endpoints
LASTECHO_ADMIN_PASSWORD=change-me

# CORS
LASTECHO_CORS_ORIGINS=http://localhost:5173

# Email — required to actually send outreach
LASTECHO_SMTP_HOST=smtp.postmarkapp.com
LASTECHO_SMTP_PORT=587
LASTECHO_SMTP_USER=<token>
LASTECHO_SMTP_PASSWORD=<token>
LASTECHO_SMTP_FROM=outreach@yourdomain.org
LASTECHO_SMTP_USE_TLS=true

# Claude AI — optional; uses template fallback if absent
ANTHROPIC_API_KEY=sk-ant-...

# Triage tuning
LASTECHO_TRIAGE_TOP_N=15         # drafts generated per sweep
LASTECHO_ESCALATE_AFTER_DAYS=7   # days before escalating to next tier
LASTECHO_ROR_CACHE_TTL_DAYS=30   # institution lookup cache TTL
```

For the client, create `client/.env.local`:

```bash
VITE_ADMIN_USER=admin
VITE_ADMIN_PASSWORD=admin123
```

## Features

### Public

- **3D Globe** — interactive visualization of ~3,300 languages per year, colored by vitality status (healthy → critical → lost)
- **Time scrubbing** — slide through 2000–2050 projections
- **Language detail** — click any language for family, speaker count, documentation level, and matched institutions
- **Rescue Queue** — real-time ranking of the most urgent languages; adjust weights (urgency, population, linguistic uniqueness) live in the browser
- **Family tree** — hierarchical view of language families

### Admin (`/about/admin`)

- **Triage sweep** — trigger ranking + AI draft generation for the top N languages
- **Outreach queue** — review, edit, approve, or reject drafted emails before they go out
- **Send & escalate** — send approved emails via SMTP; mark "no reply" to move up the institution ladder (local → continental → global)

## Architecture

### Data flow

1. Per-year language snapshots (`client/src/data/timeline_by_year/`) are bundled into the frontend and lazy-loaded on demand — no server round-trip for globe data.
2. The backend loads `api/data/languages.json` (generated by `scripts/build_data.py`) into memory at startup.
3. A triage sweep scores languages on three signals — urgency, speaker population, and linguistic uniqueness — then drafts outreach emails via Claude (or a template fallback).
4. An admin reviews the queue, edits if needed, and sends. No-reply within `ESCALATE_AFTER_DAYS` triggers escalation to the next institution tier.

### Institution ladder

For each language, the system builds a 3-tier ladder:

| Tier | Example |
|---|---|
| Local | Regional or national university / cultural center |
| Continental | Wider regional linguistic body |
| Global | UNESCO, Endangered Languages Project |

Institution matching uses offline reverse geocoding (lat/lng → country) plus the [Research Organization Registry (ROR)](https://ror.org) API, cached in SQLite.

### Key modules

```
api/
  app/
    main.py          # FastAPI setup, lifespan (data load, DB init)
    config.py        # env-based settings
    triage.py        # sweep: rank, draft, insert
    matching.py      # build institution ladder
    outreach.py      # Claude draft or template fallback
    store_db.py      # SQLite wrapper (no ORM)
    routers/         # endpoint handlers

client/
  src/
    App.tsx                      # root component, state, view routing
    components/GlobeView.tsx     # 3D visualization
    components/RescueQueue.tsx   # triage ranking UI
    components/AdminView.tsx     # admin panel
    data/timeline.ts             # per-year data loading & LRU cache
    data/triage.ts               # client-side scoring (mirrors backend)
    data/api.ts                  # backend HTTP client
```
